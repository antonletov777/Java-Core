# Структуры данных в Java: Исчерпывающее руководство

## 1. Иерархия типов и суперкласс Object
*Соответствующий класс: `ObjectContractsLecture1.java`*

В Java строгая система типов. Все ссылочные типы неявно наследуют класс `Object`, получая базовый набор методов (`getClass()`, `toString()`, `wait`/`notify`).

### Контракт `equals()` и `hashCode()`
Корректная работа хэш-коллекций (`HashMap`, `HashSet`) критически зависит от соблюдения контракта:
* **Контракт `equals`**: Должен быть рефлексивным, симметричным, транзитивным, консистентным и возвращать `false` при сравнении с `null`.
* **Контракт `hashCode`**:
    1. Если `equals` возвращает `true`, то `hashCode` объектов **обязан** совпадать. Нарушение ведет к "потере" объектов в коллекциях.
    2. Если `hashCode` равен, объекты **не обязательно** равны (коллизия).
* **По умолчанию**: `hashCode` — native метод (`identity hash code`), `equals` сравнивает ссылки (`==`).

## 2. Generics (Дженерики): Полное руководство
*Соответствующий класс: `GenericsLecture2.java`*

Дженерики (`<T>`) обеспечивают типобезопасность на этапе компиляции.

* **Type Erasure (Стирание типов)**: Дженерики существуют только для компилятора. В байткоде они стираются до границ (обычно `Object`).
* **Массивы vs Дженерики**: Массивы **ковариантны** (`String[]` подтип `Object[]`), что чревато `ArrayStoreException`. Дженерики **инвариантны**, что гарантирует безопасность.
* **Правило PECS (Producer Extends, Consumer Super)**:
    * `? extends T`: Читать можно, писать нельзя (Producer).
    * `? super T`: Писать можно, чтение деградирует до `Object` (Consumer).
* **Heap Pollution (Загрязнение кучи)**: Возникает при смешивании `varargs` и дженериков. Решается аннотацией `@SafeVarargs`.
* **Ограничения**: Нельзя использовать примитивы, инстанцировать `new T()`, создавать массивы `new T[]`.

## 3. Инструментарий, алгоритмы и фабрики (JCF)
*Соответствующий класс: `CollectionsAndArraysLecture3.java`*

Единая архитектура на базе интерфейсов `Iterable` -> `Collection` и `Map`. Объединяет возможности классов `Collections` и `Arrays`.

### Итераторы и механизмы обхода
* **Iterator**: Безопасное удаление (`remove()`) во время обхода.
* **ListIterator**: Двунаправленность, индексы и замена элементов в `List`.
* **Spliterator**: Основа Stream API. Разделяет коллекцию для параллельной обработки.
* **Fail-Fast**: Выброс `ConcurrentModificationException` при прямой модификации коллекции во время обхода.

### Утилиты и алгоритмы
* **Класс Collections**: Алгоритмы `sort` (TimSort **O(N log N)**), `binarySearch`, `shuffle`, `frequency`.
* **Checked Collections**: `checkedList()` — динамическая проверка типов для защиты от Raw Types.
* **Класс Arrays**: Утилиты для массивов, включая `deepEquals()` для матриц и мост к Stream API (`Arrays.stream()`).
* **Immutable фабрики (Java 9+)**: `List.of()`, `Set.of()`, `Map.of()`. Абсолютно неизменяемы, запрещают `null`.

## 4. Списки (List)
*Соответствующий класс: `ListImplementationsLecture4.java`*

* **ArrayList**: Динамический массив (рост x1.5).
    * *Сложность*: `get`/`set` — **O(1)**. `add` в конец — **O(1)** амортизировано. Вставка/удаление в середине — **O(N)**.
    * *Оптимизация*: `ensureCapacity(n)` предотвращает лишние реаллокации.
* **LinkedList**: Двусвязный список.
    * *Сложность*: Операции на концах — **O(1)**. Доступ по индексу — **O(N)**.
    * *Практика*: Позволяет модифицировать структуру за **O(1)** при использовании `ListIterator`.

## 5. Множества (Set)
*Соответствующий класс: `SetImplementationsLecture5.java`*

* **HashSet**: На базе `HashMap`. Базовые операции — **O(1)**.
* **LinkedHashSet**: Сохраняет порядок добавления за счет двусвязного списка.
* **TreeSet**: На базе `TreeMap`. Элементы отсортированы (**O(log N)**). Поддерживает навигацию: `lower()`, `higher()`, `subSet()`.

---

## 6. Словари (Map)
*Соответствующий класс: `MapImplementationsLecture6.java`*

* **HashMap**: Хэш-таблица (**O(1)** в среднем). При коллизиях (>=8 элементов в корзине) переходит на красно-черные деревья.
    * *Методы*: `putIfAbsent`, `getOrDefault`, `containsValue` (**O(N)**).
* **LinkedHashMap**: Поддерживает порядок вставки или порядок доступа (LRU-кэш).
* **TreeMap**: Красно-черное дерево. Ключи отсортированы (**O(log N)**). Поддерживает диапазоны `headMap`, `tailMap`.

---

## 7. Очереди (Queue и Deque)
*Соответствующий класс: `QueueDequeImplementationsLecture7.java`*

* **PriorityQueue**: Очередь с приоритетами ("Min-Heap"). `offer`/`poll` — **O(log N)**. Итератор не гарантирует порядок!
* **ArrayDeque**: Кольцевой массив. Самая быстрая замена устаревшему классу `Stack` (LIFO). Добавление/удаление с обоих концов — **O(1)**. Запрещает `null`.

---

## 8. Bitmap-структуры
*Соответствующий класс: `BitmapSetsLecture8.java`*

* **EnumSet**: Высокооптимизированная маска для `enum`. Все операции — **O(1)** (одна процессорная инструкция).
* **BitSet**: Динамический битовый вектор (`long[]`). Используется для флагов и побитовых операций `AND`, `OR`, `XOR`.
