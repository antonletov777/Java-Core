# Исчерпывающее руководство: Аннотации и Reflection API в Java

**Цель:** Освоение аннотаций и Reflection API для расширения функциональности Java-приложений и создания метапрограмм.
**Результат:** Вы научитесь создавать собственные аннотации, анализировать структуру классов в рантайме (во время выполнения) и динамически управлять объектами.

---

## 1. Введение в Аннотации (Метаданные)
*Соответствующий класс: `CustomAnnotationsLecture1.java`*

Аннотации — это специальные метки в коде. Сами по себе они ничего не делают, это просто "стикеры" с метаданными, которые могут быть прочитаны компилятором или JVM во время выполнения.

### 1.1. Базовые встроенные аннотации
* `@Override` — указывает, что метод переопределяет метод суперкласса.
* `@Deprecated` — помечает устаревший элемент (класс, метод), который не рекомендуется использовать.
* `@SuppressWarnings` — подавляет предупреждения компилятора.

### 1.2. Мета-аннотации (Аннотации для аннотаций)
Чтобы создать свою аннотацию, нужно настроить ее с помощью мета-аннотаций:
* **`@Retention`** — определяет жизненный цикл аннотации:
    * `SOURCE` — отбрасывается при компиляции.
    * `CLASS` — сохраняется в `.class` файле, но недоступна во время выполнения (по умолчанию).
    * `RUNTIME` — **самая важная для нас!** Доступна во время выполнения через Reflection API.
* **`@Target`** — указывает, куда можно "повесить" аннотацию:
    * `ElementType.TYPE` — класс, интерфейс.
    * `ElementType.FIELD` — поле класса.
    * `ElementType.METHOD` — метод.

### 1.3. Синтаксис создания
Создается с помощью ключевого слова `@interface`. Внутри можно задавать параметры (методы без тела). Если параметр один, его принято называть `value()`. Можно задавать значения по умолчанию через ключевое слово `default`.

## 2. Основы Reflection API
*Соответствующий класс: `ReflectionBasicsLecture2.java`*

**Reflection API** — это механизм, позволяющий исследовать данные о классах, интерфейсах, полях и методах во время выполнения программы, а также вызывать методы и изменять поля, даже если они приватные.

### 2.1. Получение объекта Class
Все начинается с объекта `Class<?>`, который описывает структуру типа.
* `String.class` (от типа)
* `obj.getClass()` (от объекта)
* `Class.forName("java.lang.String")` (по имени пакета и класса)

### 2.2. Анализ структуры
* `getDeclaredFields()` — получает все поля класса (включая приватные, но без унаследованных).
* `getDeclaredMethods()` — получает все методы.
* `getDeclaredConstructors()` — получает конструкторы.

### 2.3. Нарушение инкапсуляции (Магия)
Рефлексия позволяет обходить модификаторы доступа (например, `private`).
* `field.setAccessible(true)` — делает приватное поле доступным для чтения и изменения!
* `field.get(object)` — чтение значения поля у конкретного объекта.
* `field.set(object, value)` — запись значения в поле.
* `method.invoke(object, args)` — динамический вызов метода.

## 3. Метапрограммирование (Связываем всё вместе)
*Соответствующий класс: `AnnotationProcessorLecture3.java`*

Самая большая ценность этих инструментов проявляется при их совместном использовании. Мы можем написать универсальный код, который:
1. Принимает любой объект.
2. С помощью Reflection API сканирует его класс.
3. Ищет наши кастомные аннотации (например, `@Table` и `@Column`).
4. На основе значений аннотаций генерирует логику (например, SQL-запрос `CREATE TABLE` или `INSERT`).

**Ограничения и минусы:**
* **Производительность:** Рефлексия работает медленнее прямого вызова методов (хотя в современных версиях Java разница стала меньше).
* **Безопасность типов:** Ошибки выявляются не при компиляции, а в рантайме (во время выполнения).
* **Секьюрити:** В современных модульных системах (Java 9+ Project Jigsaw) доступ через рефлексию к внутренним пакетам может быть ограничен по умолчанию.
