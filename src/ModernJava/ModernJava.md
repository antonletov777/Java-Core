# Исчерпывающее руководство: Фичи новых версий Java (9 - 24)

**Цель темы:** Знакомство с новыми возможностями Java, начиная с Java 9, для использования современных инструментов и подходов в разработке.
**Результат по теме:** Ты научишься использовать `var`, `switch expressions`, текстовые блоки, `Records`, `sealed classes`, `pattern matching` и `Virtual Threads` (Project Loom).

---

## 1. Исторический контекст: Почему Java стала развиваться быстрее?

До выхода Java 9 язык развивался довольно медленно: новые "мега-релизы" выходили раз в 3-5 лет, а процесс добавления фич был сильно бюрократизирован (JCP). В то же время на платформе JVM появились более современные конкуренты, такие как Scala (2004) и Kotlin (2011), которые предложили разработчикам лаконичный синтаксис, безопасность от `null` и мощные элементы функционального программирования.

Успех этих языков показал Oracle, что сообществу нужны современные инструменты. Это привело к радикальным изменениям:
* **Новый цикл релизов:** Начиная с Java 9 (сентябрь 2017 года), новые версии выходят каждые полгода.
* **Демократизация:** OpenJDK стал главным центром разработки, куда вкладываются такие гиганты, как Google, Red Hat и Amazon.
* **Preview-фичи:** Сложные нововведения теперь обкатываются в статусе `preview`, что позволяет собирать обратную связь до их финального утверждения.

## 2. Улучшения синтаксиса и типизации
*Соответствующий класс: `ModernSyntaxLecture1.java`*

### 2.1. Ключевое слово `var` (Java 10)
Компилятор научился сам выводить тип локальной переменной на основе её значения.
* **Где работает:** Только для локальных переменных с инициализатором.
* **Где НЕ работает:** Нельзя использовать для полей класса, без начального значения (например, `var n;`), с `null` (`var empty = null;`) или в лямбдах без явного указания типа.

### 2.2. Текстовые блоки / Text blocks (Java 13)
Многострочные строки теперь можно задавать с помощью тройных кавычек `"""`.
* Компилятор автоматически высчитывает отступы.
* Больше не нужно экранировать кавычки или писать `\n` для переноса строк.

### 2.3. Switch Expressions (Java 12)
Оператор `switch` превратился в выражение, которое может возвращать значение.
* Используется новый синтаксис с `->`.
* Предотвращает ошибку "сквозного выполнения" (fall-through), из-за которой раньше приходилось писать `break` в каждом `case`.
* Можно группировать условия: `case MONDAY, FRIDAY, SUNDAY -> 6;`.

## 3. Моделирование данных: Records и Sealed Classes
*Соответствующий класс: `DataModelingLecture2.java`*

### 3.1. Records (Java 16)
`Record` — это новый вид класса, предназначенный исключительно для хранения неизменяемых (immutable) данных.
* Автоматически генерирует: конструктор, геттеры (без префикса `get`), методы `equals()`, `hashCode()` и `toString()`.
* Класс неявно является `final`.
* Можно добавлять валидацию в компактный конструктор или создавать дополнительные конструкторы.

### 3.2. Sealed Classes (Java 15)
"Запечатанные" классы позволяют автору строго контролировать иерархию наследования.
* С помощью ключевого слова `permits` перечисляются классы, которым разрешено наследоваться от данного класса или интерфейса.
* Это защищает архитектуру от появления непредусмотренных подклассов и делает проверки типов (в том же `switch`) исчерпывающими.

## 4. Pattern Matching (Сопоставление с шаблоном)
*Соответствующий класс также: `DataModelingLecture2.java`*

Pattern matching — это механизм проверки значения на соответствие определенному шаблону, пришедший из функциональных языков. В Java он внедряется поэтапно:

### 4.1. Pattern Matching для `instanceof` (Java 14)
Позволяет объявить переменную нужного типа прямо внутри проверки `instanceof`, избавляя от необходимости писать явное приведение типов.
* Также можно добавлять дополнительные условия сразу в `if`: `if (shape instanceof Rectangle r && r.length() > 5)`.

### 4.2. Pattern Matching в `switch` и Record Patterns (Java 19+)
Теперь `switch` может принимать объекты любых типов и проверять их через паттерны.
* **Деструктуризация (Destructuring):** Позволяет прямо в условии разобрать `Record` на составные части: `case Location(String name, GPSPoint gps) -> ...`.

## 5. Многопоточность и Сеть: Virtual Threads & HTTP Client
*Соответствующий класс: `ConcurrencyAndNetworkLecture3.java`*

### 5.1. Новый HTTP Client (Java 11)
Заменил устаревший `HttpURLConnection`. Он удобный, поддерживает HTTP/2 и WebSockets, а также легко работает как в синхронном, так и в асинхронном режиме (через `CompletableFuture`).

### 5.2. Виртуальные потоки / Virtual Threads (Java 21 - Project Loom)
Это легковесные потоки, которые управляются самой JVM, а не операционной системой.
* **Суть проблемы:** Раньше на каждый запрос создавался отдельный поток ОС (модель thread-per-request). Потоки ОС тяжелые, поэтому приложение не могло держать миллионы активных соединений. Приходилось писать сложный асинхронный код с коллбэками.
* **Решение:** Виртуальный поток не привязан жестко к потоку ОС. Если виртуальный поток ждет ответа от базы данных (блокирующий I/O), JVM "отвязывает" его, и поток ОС берет в работу другой виртуальный поток.
* **Итог:** Мы снова можем писать простой, понятный синхронный код, но при этом приложение легко держит миллионы одновременных виртуальных потоков.
